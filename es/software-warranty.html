Asegurar que la garantía del software dure mucho más

A diferencia de los objetos físicos, el software no se erosiona con el tiempo. Si pagas por una aplicación para smartphone, no debería romperse a los pocos días. Al igual que cualquier otro producto que compres, el software debería tener una garantía; una que dure mucho más.

Cómo funciona

¿Por qué garantía de software?

Cualquier producto comprado en la Unión Europea tiene una garantía limitada: si el producto se rompe o se comporta mal durante ese periodo, puede hacer que le reparen o le sustituyan el producto, o le devuelvan el dinero.

La duración de la garantía suele depender del tipo de producto. Es razonable que un smartphone dure al menos 2 años, pero un plátano probablemente no dure tanto. Para el software, sin embargo, no hay una duración determinada de cuánto debería durar "razonablemente" el software si lo compras.

¿Cómo podemos solucionar esto?

Si compras una aplicación en una tienda de aplicaciones, debería funcionar - y si no funciona o si se vuelve inutilizable después de un tiempo, deberías recuperar tu dinero.

Deberíamos empezar a tratar las aplicaciones y el software como productos: si pagas por él, debería haber una garantía. Los expertos deberían determinar cuánto tiempo debería funcionar razonablemente una aplicación, y luego esas garantías deben ser legisladas.

Nadie sabe

Cuánto dura la garantía del software? 🔖 Resulta que eso es algo que nadie sabe.

He llamado a la autoridad holandesa en materia de garantías, a la autoridad técnica, a un proveedor de software y a la ventanilla jurídica para los ciudadanos, y ninguno de ellos ha sido capaz de darme una respuesta. 🙈

Me aconsejaron que consultara a un abogado porque "tal vez ellos sepan".
